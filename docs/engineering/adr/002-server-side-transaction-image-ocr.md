# ADR-002: 거래 이미지 OCR의 서버 이전

## Status

Accepted

## Context

- 브라우저의 Tesseract.js는 기기 성능에 따라 처리 시간이 달라지고, 모바일에서 모델 다운로드와 메모리 사용 부담이 큽니다.
- 어두운 결제 내역 화면은 원본만 인식할 때 상태바 숫자, 아이콘, 줄바꿈된 상호명이 거래 데이터에 섞입니다.
- 외부 OCR 서비스는 정확도 개선 가능성이 있지만 비용, credential 관리, 이미지의 외부 전송 검토가 먼저 필요합니다.

## Options

- 브라우저 Tesseract.js를 유지하고 프론트엔드에서 전처리합니다.
- 백엔드 컨테이너에서 Native Tesseract와 고정된 한국어/영어 모델을 실행합니다.
- 외부 OCR API로 이미지를 전송합니다.

## Decision

- 이미지 업로드, 전처리, OCR, 거래 후보 파싱을 백엔드에서 수행합니다.
- Docker 이미지에 Tesseract 5와 commit/hash가 고정된 `tessdata_best`의 `kor+eng` 모델을 포함합니다.
- 원본 RGB와 대비 강화·반전 이미지를 기본 인식하고 후보별 결과를 결합합니다. 두 결과가 비어 있거나 후보 수가 다를 때만 이진화 이미지를 추가 인식합니다.
- `시간 + #결제수단 + 잔액` 행이 감지되는 목록형 화면은 PSM 11 희소 레이아웃 OCR을 한 번 더 실행합니다. 금액 순서를 기준으로 기본 OCR의 상호명과 희소 OCR의 누락 거래를 결합하고 잔액 행은 후보에서 제외합니다.
- 업로드는 PNG/JPEG, 10MB 이하로 제한하고, 최소/최대 해상도와 전체 픽셀 수를 검증합니다.
- 원본과 전처리 이미지는 임시 디렉터리에서만 사용하고 요청 종료 시 삭제합니다.
- 직접 입력 텍스트 preview API는 별도로 유지합니다.

## Consequences

- 프론트엔드에서 Tesseract.js와 OCR 모델을 다운로드하지 않습니다.
- 백엔드 런타임에 Tesseract 실행 파일과 `kor`, `eng` 모델이 필요합니다. Docker 개발 환경에는 포함되며 호스트 직접 실행 시 별도 설치해야 합니다.
- 일반 샘플 기준 처리 시간은 이미지당 약 7~9초입니다. 희소 레이아웃 보조 OCR이 실행된 샘플은 약 11초였으며, 서버 CPU 사용량과 동시 요청 제한은 운영 전 추가 관찰이 필요합니다.
- 전처리와 후보 결합으로 일반적인 상태바·아이콘·줄바꿈 잡음을 줄이지만, 원본 글자가 모든 변형에서 같은 잘못된 라틴 문자로 인식되는 경우에는 해결되지 않습니다. 현재 `긱스타PC` 샘플은 이 경우라 좁은 파서 보정이 남아 있습니다.
- 외부 OCR은 샘플 범위가 넓어져 현재 정확도나 처리 시간이 수용 기준을 충족하지 못할 때 다시 평가합니다.
