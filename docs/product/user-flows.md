# User Flows

## 1. First Login

1. 사용자가 Kakao로 로그인합니다.
2. 서버가 사용자를 생성하거나 기존 사용자를 찾습니다.
3. 사용 가능한 장부가 없으면 기본 개인 장부를 생성합니다.
4. 최근 사용 장부가 있으면 해당 장부로 진입합니다.
5. 최근 사용 장부에 접근할 수 없으면 기본 개인 장부로 fallback합니다.

## 2. Ledger Switching

1. 사용자가 상단 장부 선택기를 엽니다.
2. 개인 장부와 공동 장부가 하나의 리스트에 표시됩니다.
3. 사용자가 장부를 선택합니다.
4. 서버가 마지막 사용 장부를 저장합니다.
5. 대시보드와 하위 화면이 선택 장부 기준으로 갱신됩니다.

## 3. Transaction Recording

1. 사용자가 FAB 또는 거래 입력 영역을 엽니다.
2. 사용자가 금액, 일자, 카테고리, 메모, 결제자를 입력합니다.
3. 서버가 현재 장부와 월 기준으로 거래를 저장합니다.
4. 대시보드, 월 거래 목록, 통계가 갱신됩니다.
5. 마감된 월에는 거래를 생성, 수정, 삭제할 수 없습니다.

## 4. Quick Transaction

1. 사용자가 대시보드의 빠른 입력 항목을 선택하거나 문장을 입력합니다.
2. 금액, 카테고리, 메모 등 후보 값이 입력 시트에 채워집니다.
3. 사용자가 값을 확인하고 필요하면 보정합니다.
4. 거래가 생성됩니다.

## 5. Monthly Budget Setup

1. 사용자가 월 설정 화면에 진입합니다.
2. 사용자가 월 전체 예산을 설정합니다.
3. 사용자가 카테고리별 예산과 멤버별 할당을 설정합니다.
4. 서버가 해당 장부/월 기준 설정을 저장합니다.
5. 대시보드와 통계가 예산 기준으로 계산됩니다.

고정비가 있으면 월 예산을 처음 열 때 해당 카테고리 예산의 기본값으로 미리 채워집니다. 고정비 관리 자체는 거래를 생성하지 않습니다.

## 6. Monthly Closing

1. 사용자가 월 설정 화면에서 마감을 실행합니다.
2. 서버가 해당 월 상태를 closed로 변경합니다.
3. 닫힌 월은 예산과 거래 변경, 반복 거래 생성을 차단합니다.
4. 필요하면 사용자가 재오픈합니다.

## 7. Invitation

1. 공동 장부 관리자가 초대 대상을 검색하거나 초대 링크를 생성합니다.
2. 초대받은 사용자는 pending invitation 또는 link page에서 초대를 확인합니다.
3. 사용자가 초대를 수락하면 장부 멤버가 됩니다.
4. 사용자가 거절하면 초대가 종료됩니다.

## 8. Recurring Transaction

1. 사용자가 반복 거래 템플릿을 생성합니다.
2. 서버가 생성 예정 거래를 조회합니다.
3. 사용자가 생성 실행을 요청합니다.
4. 서버가 중복 생성을 방지하며 거래를 생성합니다.

## 9. Transaction Import

1. 사용자가 거래 이미지 또는 텍스트를 업로드합니다.
2. 서버가 OCR 또는 파서를 통해 거래 후보를 만듭니다.
3. 사용자가 preview에서 type, 금액, 날짜, 카테고리, 메모를 확인하고 수정합니다.
4. 확인한 후보를 일반 거래 생성 규칙으로 저장합니다.

## 10. Monthly Settlement

1. 공동 장부 사용자가 월 설정에서 정산 결과를 확인합니다.
2. 서버가 멤버별 할당 비율과 실제 결제액으로 송금 방향과 남은 금액을 계산합니다.
3. 사용자가 일부 또는 전체 송금을 기록합니다.
4. 송금 기록을 취소하면 남은 정산 금액이 다시 계산됩니다.

## 11. Notification

1. 직접 초대, 월 마감, 예산 초과 이벤트가 발생합니다.
2. 관련 사용자의 알림함에 알림이 생성됩니다.
3. 사용자는 알림을 개별 또는 전체 읽음 처리합니다.

## 12. Ledger Management

1. 공동 장부 소유자는 장부 이름 변경, 멤버 내보내기, 장부 보관을 수행할 수 있습니다.
2. 일반 멤버는 공동 장부에서 스스로 탈퇴할 수 있습니다.
3. 보관하거나 탈퇴한 장부가 현재 장부라면 접근 가능한 장부로 전환됩니다.

## 13. Category Management

1. 사용자가 거래 입력의 `카테고리 관리` 또는 앱 보조 메뉴에서 카테고리 관리 화면으로 이동합니다.
2. 사용자가 필요하면 통계 대분류를 만듭니다.
3. 사용자가 대분류를 선택해 세부 카테고리를 만들거나 기존 카테고리의 이름·대분류를 수정합니다.
4. 사용자가 미사용 카테고리를 삭제할 수 있으며, 거래·예산·반복 거래에 연결된 카테고리는 삭제할 수 없습니다.
5. 거래 입력에는 세부 카테고리 이름만 표시하고, 통계와 대시보드에서는 대분류별로 합산해 표시합니다.
